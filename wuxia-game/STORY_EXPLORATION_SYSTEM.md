# 🎮 剧情与探索交替系统

## 🎯 设计理念

**剧情不再是连续的线性流程，而是与地图探索交替进行。**

玩家的体验：
```
开场剧情 → 自由探索 → 走到新地点 → 触发剧情 → 自由探索 → ...
```

---

## 📍 工作原理

### 1. 地图位置绑定剧情

在 `worldMap.ts` 中配置：

```typescript
const CELL_META: Record<string, CellMeta> = {
  '2,2': { 
    name: '青云山', 
    scenarioId: 'start',      // 绑定剧情节点
    triggerOnce: true         // 只触发一次
  },
  '3,2': { 
    name: '山门出口', 
    scenarioId: 'mountain_exit' 
  }
}
```

### 2. 剧情节点设计

在 `chapter1.json` 中，每个节点的选择都指向自己：

```json
{
  "start": {
    "text": ["开场文字..."],
    "choices": [
      {
        "text": "【踏上旅途】",
        "next": "start"  // ← 指向自己，表示结束过场
      }
    ]
  }
}
```

**关键逻辑：**
- 当 `next === 当前节点ID` 时，点击后关闭过场
- 玩家回到地图，可以自由移动
- 走到新地点时，触发新的剧情

## 🎬 完整游戏流程示例

### 第1步：游戏开始
```
玩家点击"开始游戏"
↓
触发 start 节点（过场）
显示："洪武二十年，江南烟雨..."
↓
玩家点击【踏上旅途】
↓
关闭过场，显示地图
玩家在青云山 (2,2) 位置
```

### 第2步：自由探索
```
玩家使用方向键移动
↓
从 (2,2) 移动到 (3,2)
↓
触发 mountain_exit 节点（过场）
显示："你走下青云山，来到山脚..."
↓
玩家点击【继续前行】
↓
关闭过场，继续探索
```

### 第3步：遇到事件
```
玩家继续移动
↓
走到 (1,3) 密林入口
↓
触发 forest_entrance 节点（过场）
显示："林中传来打斗声和呼救声！"
↓
玩家选择：
  【前去查看】→ 跳转到 forest_rescue（战斗）
  【绕道而行】→ 关闭过场，继续探索
```

### 第4步：战斗
```
如果选择【前去查看】
↓
跳转到 forest_rescue 节点
↓
触发战斗
↓
战斗结束后：
  胜利 → rescue_win 节点
  失败 → rescue_lose 节点
↓
看完结果剧情
↓
关闭过场，继续探索
```

## 🔧 节点设计规范

### 规范1：结束过场的节点
```json
{
  "node_id": {
    "text": ["剧情文字"],
    "choices": [
      {
        "text": "【按钮文字】",
        "next": "node_id",  // ← 指向自己
        "effects": []        // 可选：应用效果
      }
    ]
  }
}
```

### 规范2：跳转到其他节点
```json
{
  "node_id": {
    "text": ["剧情文字"],
    "choices": [
      {
        "text": "【选项A】",
        "next": "other_node"  // ← 跳转到其他节点
      },
      {
        "text": "【选项B】",
        "next": "node_id"     // ← 结束过场
      }
    ]
  }
}
```

### 规范3：战斗节点
```json
{
  "node_id": {
    "text": ["战斗前的文字"],
    "battle": {
      "enemyId": "bandit",
      "canFlee": true,
      "winNext": "win_node",   // 胜利后的节点
      "loseNext": "lose_node"  // 失败后的节点
    }
  }
}
```

## ✨ 优势

### 1. 自由度高
- 玩家可以选择探索路线
- 不是被剧情牵着走
- 可以跳过某些地点

### 2. 节奏好
- 剧情和探索交替
- 不会连续看很久剧情
- 不会长时间没有剧情

### 3. 易扩展
- 添加新地点：在地图配置中添加
- 添加新剧情：在 chapter1.json 中添加节点
- 两者独立，互不影响

### 4. 重复可玩性
- 每次可以选择不同路线
- 不同选择有不同结果
- 可以多周目体验

## 📝 添加新剧情的步骤

### 步骤1：在 chapter1.json 添加节点
```json
{
  "your_new_node": {
    "id": "your_new_node",
    "atmosphere": "normal",
    "text": ["你的剧情文字"],
    "choices": [
      {
        "text": "【确定】",
        "next": "your_new_node"
      }
    ]
  }
}
```

### 步骤2：在 worldMap.ts 绑定位置
```typescript
const CELL_META: Record<string, CellMeta> = {
  // ... 其他配置
  '5,5': { 
    name: '新地点', 
    scenarioId: 'your_new_node',
    triggerOnce: true 
  }
}
```

### 步骤3：在 GameViewOptimized.vue 添加到节点集合
```typescript
const ALL_SCENARIO_IDS = new Set([
  // ... 其他节点
  'your_new_node'
])
```

完成！玩家走到 (5,5) 时就会触发你的新剧情。

## 🎯 与原版的区别

### 原版（线性剧情）
```
开始 → 节点1 → 节点2 → 节点3 → ... → 节点15 → 自由探索
```
- ❌ 需要看完所有剧情才能探索
- ❌ 剧情冗长
- ❌ 缺少自由度

### 新版（交替模式）
```
开始 → 探索 → 节点A → 探索 → 节点B → 探索 → ...
```
- ✅ 立即可以探索
- ✅ 剧情分散在各个地点
- ✅ 高度自由

## 🐛 常见问题

### Q: 为什么选择要指向自己？
A: 这是一个约定，表示"这个节点是终点，点击后关闭过场"。

### Q: 如果不想触发剧情怎么办？
A: 不要走到那个地点就行了。所有剧情都是可选的（除了开场）。

### Q: 可以重复触发剧情吗？
A: 默认 `triggerOnce: true`，只触发一次。如果想重复触发，设为 `false`。

### Q: 战斗后会自动关闭过场吗？
A: 不会。战斗后会跳转到 `winNext` 或 `loseNext` 节点，需要在那些节点中设置关闭过场的选项。

## 💡 设计建议

1. **开场简短** - 只介绍必要信息
2. **地点剧情独立** - 每个地点的剧情可以独立理解
3. **选择有意义** - 不同选择应该有不同结果
4. **战斗合理** - 不要在每个地点都放战斗
5. **奖励明确** - 探索地点应该有奖励（物品、声望等）
